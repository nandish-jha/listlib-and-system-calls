Overview:
---------
This document outlines the test plan for the linked list implementation, 
testing both basic and advanced functionality across multiple list instances.
The test cases verify the correctness of list operations including creation,
manipulation, and cleanup.

Test Environment:
----------------
- Compiler: gcc with -std=gnu90
- Test Files: mytestlist.c, list.h
- Related Files: list_adders.c, list_movers.c, list_removers.c

Test Execution:
--------------
1. Compile the test program:
   make clean && make 
   OR 
   bash partB.bash partC

2. Run the test program (Not required if running bash script):
   ./mytestlist

3. Monitor test progress:
   - Program tracks number of passed tests with tests_passed counter
   - Each successful test increments the counter (total of 15 tests)
   - Success/failure messages are printed for each test
   - Final test results should show 15/15 tests passed for complete success

4. Test Success Validation:
   - All 15 tests must pass
   - tests_passed counter should reach 15 by end of execution
   - No error messages should be displayed
   - All operations should maintain list integrity

Expected Results:
---------------
- All test cases should pass without errors
- The orderring and combination of test case execution itself exercises
    the data structure in several ways.
- Memory Pool should be properly managed without any accidental overlap
- List operations should maintain data integrity
- Multiple list instances should work independently 
    no interference with NODEs from other lists
- Concatenation should properly combine lists and free up the 2nd list

Notes:
------
- Test cases are designed to verify both basic and simple edge cases
- Each operation is verified for correct return values
- List state is validated after each operation
- Memory management is tested throughout the process
