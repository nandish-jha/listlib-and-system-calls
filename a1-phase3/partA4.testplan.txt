Part A4 Comprehensive Test Plan
=======================================

Overview:
---------
This test plan provides exhaustive testing coverage for Part A4, which
implements a multi-process solution using UNIX processes (fork() system
call) to compute squares using a recursive algorithm. The implementation
creates multiple child processes using fork(), each calling Square(N) for
values 1 to N until a deadline is reached, with inter-process communication
through shared memory.

Implementation Details:
----------------------
- Uses UNIX fork() system call for process creation
- Main process creates multiple child processes directly
- Inter-process communication via shared memory (shmget, shmat, shmdt)
- Signal handling for process termination (SIGUSR1, kill())
- Global volatile flag 'keep_running' controls execution per process
- Process-local storage for current_thread identification
- Timing measurements using gettimeofday() for microsecond precision
- Memory allocation for process IDs and shared memory management
- Process synchronization using waitpid() for cleanup
- Uses sleep() for timing delays (POSIX sleep function)
